Work with Multiple Assessments

 
Using multiple assessments, called Aggregation in CSET, means that you want to work with two or more assessments rather than just create a new one. To begin a new Aggregation, select the Trend or Compare option in the Tools menu. Previous Trend and Compare files are saved in a list. You can access them when you select Trend or Compare.
 
Figure: Saved Comparison page
 
Selecting New Comparison or New Trend will open a Details screen shown below.
 
Figure: Comparison Details screen
 
There are two types of aggregation; Trend and Compare.
 
  • Trend shows cumulative changes to a facility over time by analyzing several assessments of the same facility completed, say, annually. The analysis result is a report.
 
  • Compare looks at multiple assessments to determine their strengths and weaknesses, which creates a summary compliance report. This would be helpful for a facility consisting of several sites, each with its own completed assessment. The analysis result is a report.
 

Load Assessment Files:

 
To begin, add the assessment files to be analyzed. Click the Select Assessments button. In the Select Assessments popup, select the desired assessment files. At least two files must be present in order for the aggregation function to be performed. It is recommended that no more than five assessments be analyzed at a time. The figure below shows the screen with files added and the compatibility score.
 
Figure: Select Assessments popup
 
The compatibility estimates reflect the similarity of the standards addressed by the loaded assessments. It is not an estimate of how alike the answers are among the assessments. A low compatibility percentage is defined by having little to minimal overlap in questions or requirements contained within the loaded standards between the assessments.
 
Click "OK" to see the Assessment table.
 
Figure: Assessments table
 
1) This table lists the assessments to be included in the aggregation. The parts of the table are:
 
Alias: The Alias is a short nickname used to quickly identify the assessment in the analysis charts. The prepopulated name may be changed by clicking within the box and typing the desired alias.
 
Assessment Name: The user-entered name supplied under the Information tab during the original assessment.
 
Assessment Date: The user-entered date supplied under the Information tab during the original assessment.
 
2) The standards table lists all the standards used by the included assessments and indicates which assessments used which standards. It uses the Alias to identify the assessments.
 
 

Perform the Analysis

 
Once the assessment file table is complete, click the Next button to begin the analysis.
 
For Trend and Compare:
After clicking next the results screen will load. Compare will have multiple sub-headings of information to review. Click Generate Report to create an HTML report of the results which will include the text entered under the aggregation results and the Mode and SAL from the Default assessment.
